# UI 需求文档

## 1. 概述

### 1.1 文档目的

本文档定义 Krasis 笔记应用的用户界面需求，包括页面布局、功能交互、组件规范等，用于指导前端开发实现。

### 1.2 目标平台

| 平台 | 技术栈 | 最低版本 |
|------|--------|----------|
| iOS | Flutter | iOS 12.0+ |
| Android | Flutter | Android 5.0+ (API 21) |
| Web | Vue 3 + Vite + TypeScript | 现代浏览器 |

### 1.3 技术选型

| 平台 | 框架 | 构建工具 | 语言 | 状态管理 | 路由 |
|------|------|----------|------|----------|------|
| 移动端 | Flutter | - | Dart | Provider | go_router |
| Web | Vue 3 | Vite | TypeScript | Pinia | Vue Router |

**Web 端依赖：**

```json
{
  "dependencies": {
    "vue": "^3.4",
    "vue-router": "^4.2",
    "pinia": "^2.1",
    "axios": "^1.6",
    "element-plus": "^2.5",
    "@element-plus/icons-vue": "^2.3"
  }
}
```

**Web 端组件库：** TDesign Vue（腾讯企业级 UI 组件库）

### 1.4 设计风格

- **设计语言**：TDesign 设计系统
- **主题**：支持浅色/深色模式切换
- **设计原则**：简洁、直观、一致性

#### 3.4.6 内容存储与编辑规则（Markdown 作为存储格式）

为统一后端存储与多端展示，笔记内容**以 Markdown 作为存储格式**，但编辑体验为**所见即所得（WYSIWYG）**，用户在编辑时不需要看到 Markdown 语法符号。

**规则：**

- **标题自动生成**：如果用户未输入标题（标题为空或全空白），保存时客户端将使用“内容首句”自动生成标题，并写入 `notes.title`。
  - **首句定义**：取内容的第一段非空文本，按句号/问号/感叹号/换行分句后取第一句。
  - **长度限制**：标题最大长度为 **80 个字符**（超出部分截断），以保证在列表中可读（后端字段上限为 500）。
- **编辑器展示**：编辑区以富文本方式呈现（加粗/标题/列表等样式），UI **不展示** `#`、`**`、`- `、``` 等 Markdown 符号。
- **保存行为**：保存时将编辑器当前内容**导出为 Markdown 字符串**写入 `notes.content`。
- **粘贴行为（关键）**：如果用户粘贴的内容本身是 Markdown 文本（例如包含 `#`、`**bold**`、```code``` 等），客户端**不做“转 Markdown”/清洗/重写**，按原文本插入并保存（即“原样存储”）。
  - 说明：粘贴 Markdown 视为普通文本输入，不进行自动识别与语法转换。
- **加载行为**：打开笔记时，从 `notes.content` 读取 Markdown，并在编辑器中以富文本形态展示。

**实现要求：**

- **Vue Web**：使用 WYSIWYG 编辑器，保存导出 Markdown；禁用“粘贴时将 Markdown 自动转换为富文本/重写 Markdown”的功能。
- **Flutter**：使用富文本编辑器（如 Quill），保存导出 Markdown；粘贴 Markdown 文本时按普通文本插入并原样保存。

#### 3.4.7 编辑工具栏（按截图实现）

编辑器工具栏需提供与截图一致的编辑能力（布局可按平台适配，但功能项必须齐全）。

**工具栏功能项：**

- **插入**
  - 插入图片（URL 或上传能力，至少支持 URL）
  - 语音输入（录音上传为语音文件）
  - （可选）插入链接
- **段落样式**
  - 正文（Paragraph）
  - 标题 H1 / H2 / H3
- **字体**
  - 默认字体（系统字体）
  - （可选）Serif / Monospace 等
- **字号**
  - 12 / 14 / 16 / 18 / 20 / 24（至少包含 14）
- **文本样式**
  - 加粗（B）
  - 斜体（I）
  - 下划线（U）
  - 删除线（S）
- **颜色**
  - 文字颜色（A）
  - 高亮/背景色（Marker）
- **列表**
  - 有序列表
  - 无序列表
- **对齐**
  - 左对齐 / 居中 / 右对齐（可选：两端对齐）
- **更多**
  - 代码块（或其它扩展能力入口）

**交互规则：**

- 工具栏按钮对当前选区生效；无选区时对光标所在位置的输入生效。
- 插入图片/颜色/高亮涉及参数输入时，优先使用平台原生对话框或弹层。
- 语音输入：
  - 点击开始录音；再次点击停止并上传
  - **语音文件名默认用当前日期与时间组成**，格式：`YYYYMMDD_HHMMSS.<ext>`（例：`20260425_221915.webm` / `20260425_221915.m4a`）
  - 上传成功后在正文插入一条引用（例如：`[语音 20260425_221915.m4a](file:<file_id>)`），方便后续播放/下载能力接入
- 编辑器以所见即所得方式显示，不展示 Markdown 符号；保存时导出为 Markdown。

### 4.1 Web 端组件（Vue + TDesign）

#### 按钮 (t-button)

| 类型 | TDesign variant | 用途 |
|------|-----------------|------|
| Primary | `theme="primary"` | 主要操作 |
| Default | `theme="default"` | 次要操作 |
| Text | `variant="text"` | 辅助操作 |
| Danger | `theme="danger"` | 危险操作 |
| Loading | `loading={true}` | 加载状态 |

```vue
<t-button theme="primary">主要操作</t-button>
<t-button theme="danger" loading>删除中</t-button>
```

#### 输入框 (t-input)

| 状态 | TDesign 属性 | 样式 |
|------|-------------|------|
| Default | - | 灰色边框 |
| Focused | `status="undefined"` | 蓝色边框 |
| Error | `status="error"` | 红色边框 + 错误提示 |
| Disabled | `disabled` | 灰色背景 |

#### 卡片 (t-card)

```vue
<t-card title="卡片标题" hover>
  卡片内容
</t-card>
```

#### 表格 (t-table)

```vue
<t-table :data="tableData" :columns="columns" row-key="id">
  <template #status="{ row }">
    <t-tag :theme="row.status === 'pending' ? 'warning' : 'success'">
      {{ row.statusText }}
    </t-tag>
  </template>
