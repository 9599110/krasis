# 系统概览

## 1. 项目定位

**NoteKeeper** - 企业级智能笔记系统，支持 AI 知识库、全文检索、实时协同编辑的跨平台笔记解决方案。

## 2. 核心目标

| 目标 | 描述 |
|------|------|
| 高性能 | API 响应时间 95% < 200ms |
| 安全可靠 | OAuth 2.0 + JWT 认证，细粒度 RBAC 权限控制 |
| 智能辅助 | 基于 RAG 的 AI 问答，智能检索 |
| 实时协同 | WebSocket 实时协同编辑，支持多端同时在线 |
| 跨平台 | iOS、Android、Web、桌面端 |

## 4. 技术栈总览

### 4.1 服务端

| 层次 | 技术 | 说明 |
|------|------|------|
| 语言 | Go 1.21+ | 高性能、协程并发 |
| Web 框架 | Gin / Echo | RESTful API |
| WebSocket | gorilla/websocket | 实时协同 |
| 数据库 | PostgreSQL 15+ | 关系型数据 |
| 缓存 | Redis 7.x | 会话、队列、锁 |
| 全文检索 | Elasticsearch 8.x | 笔记搜索 |
| 向量数据库 | Qdrant | AI 语义检索 |
| 对象存储 | MinIO | 文件存储 |
| AI | LangChain Go / 本地 LLM | RAG 知识库 |
| 任务队列 | Asynq | 异步任务 |
| 容器化 | Docker + K8s | 部署运维 |

### 4.2 客户端

| 平台 | 技术 | 说明 |
|------|------|------|
| iOS | Flutter 3.16+ | 跨平台框架 |
| Android | Flutter | 跨平台框架 |
| Web | Flutter Web | 响应式设计 |
| Desktop | Flutter Desktop | Windows/macOS/Linux |
| 状态管理 | Riverpod / Bloc | 状态管理 |
| 网络 | Dio + Retrofit | HTTP 客户端 |
| WebSocket | web_socket_channel | 实时通信 |
| 安全存储 | flutter_secure_storage | 敏感数据 |

## 7. 安全架构

### 7.1 认证流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        认证流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. OAuth 2.0 登录                                               │
│     ├─ GitHub / Google 第三方授权                                │
│     ├─ 获取用户信息，自动创建本地账号                            │
│     └─ 生成 JWT + Session                                       │
│                                                                  │
│  2. JWT Token 结构                                               │
│     ├─ Header: alg: HS256, typ: JWT                             │
│     ├─ Payload: user_id, role, session_id, exp, iat             │
│     └─ Signature: HMAC-SHA256(Header.Payload, Secret)          │
│                                                                  │
│  3. Session 管理 (Redis)                                         │
│     ├─ Key: session:{session_id}                                │
│     ├─ Value: {user_id, device_info, last_active}              │
│     └─ TTL: 7 days                                              │
│                                                                  │
│  4. 多端支持                                                     │
│     ├─ 同一用户可拥有多个 session_id                            │
│     ├─ user_sessions:{user_id} → Set<session_id>               │
│     └─ 支持远程设备下线                                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 8. 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| API 响应时间 | 95% < 200ms | 不含 AI 调用 |
| AI 首字延迟 | < 3s | 流式输出 |
| 并发用户 | 1000+ 在线 | 水平扩展 |
| 笔记保存 | < 100ms | 乐观锁操作 |
| 文件上传 | 支持 100MB | 预签名直传 |
| 搜索响应 | < 500ms | ES 查询 |
